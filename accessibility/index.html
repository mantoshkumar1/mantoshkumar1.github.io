Accessibility

Designed for different ways of using the web

This website aims to remain understandable and operable with a keyboard, screen reader, zoom, high-contrast settings, reduced motion, and color-vision differences.

What the website supports

  • Semantic headings, landmarks, page titles, and a skip-to-content link.
  • Keyboard navigation with visible focus and no pointer-only action.
  • Text links that are identifiable without relying on color alone.
  • Visitor-selectable Auto, Light, Dark, Soft, and High contrast appearances, with explicit choices saved on the device.
  • System preferences for reduced motion, increased contrast, reduced transparency, and forced colors.
  • Ask Mantosh dialog focus management, keyboard dismissal, and a quieter screen-reader announcement after streaming completes.
  • Responsive text and layouts that reflow without horizontal page scrolling.

Language and translation

English is the authoritative language for the website. The pages use standard language metadata and selectable text so visitors can use translation built into their browser. The website does not infer language from location or send page content to a third-party translation widget.

Audio and video

The website currently contains no audio or video, so captions and transcripts are not required. If time-based media is added later, captions or transcripts will be required before publication.

Résumé formats

The downloadable PDF preserves the intended visual layout but is not currently a tagged PDF. The HTML résumé summary and full experience page provide the accessible web alternatives.

External services

LinkedIn, GitHub, and externally hosted project demonstrations have their own accessibility behavior. Their interfaces are outside this website’s control.

Report a barrier

If something prevents you from reading, navigating, or contacting Mantosh, email mantoshk234@gmail.com or use the contact page. Include the page and what you were trying to do.

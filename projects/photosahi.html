Project Case Study

PhotoSahi

Privacy-first passport and visa photo generation running entirely inside the browser.

By Mantosh Kumar · Updated July 11, 2026

Live Demo GitHub

Problem

Creating passport and visa photos is frustrating. Most online tools require users to upload sensitive images to remote servers, which creates privacy concerns and leaves too much room for error. Official photo requirements are also difficult to satisfy without a clear, guided workflow.

Solution

PhotoSahi was built to remove that friction. The experience runs completely in the browser, uses HTML5 Canvas for local processing, and requires no backend. Photos are processed locally, never uploaded, and the workflow is optimized for passport and visa photo preparation.

Technical Highlights

Client-side Processing

Image transformations happen locally in the browser, which keeps the experience fast and responsive.

Privacy First

Nothing needs to be uploaded to a remote service, which makes the workflow more trustworthy.

Zero Backend Cost

By avoiding server-side image handling, the system remains simple and inexpensive to operate.

Fast Image Processing

The use of canvas primitives keeps transformations lightweight for the end user.

Responsive UI

The interface was designed to work smoothly across desktop and mobile environments.

Open Source

The project is intended to be approachable and transparent for learning and reuse.

Engineering Decisions

Why browser-only?

It allows the tool to be simple, immediate, and private without introducing operational complexity.

Why no backend?

A backend would add latency, cost, and trust concerns that were not necessary for the core use case.

Performance

The implementation focuses on efficient canvas operations and careful image handling to stay responsive.

Privacy and trade-offs

Keeping everything local improves privacy, but it also means the browser must handle more of the complexity and resource use.

Lessons Learned

  • Image processing requires careful attention to quality, cropping, and edge cases.
  • The Canvas API is powerful, but browser compatibility and performance must be considered from the beginning.
  • User experience matters as much as technical correctness when the task is highly procedural.
  • Small workflow improvements can make a big difference for users preparing official documents.

Future Improvements

  • More precise compliance presets for different countries and document types.
  • Improved background adjustment controls and previewing.
  • Better accessibility and guidance for first-time users.
  • Additional export options and better image quality tuning.

GTT Trigger Price Calculator

A small Streamlit application that turns documented trigger-distance and rounding assumptions into suggested buy and sell prices.

Inspect source on GitHub ↗ All systems

Problem

Trigger-order interfaces reject values that fall outside their accepted distance or price increment. The application makes its assumptions explicit, calculates candidate values, and optionally converts between order amount and share count.

Current implementation

  • Accepts a positive current stock price and a configurable rounding multiple.
  • For prices at or above ₹50, the code applies a 0.256% distance buffer; below ₹50, it applies ₹0.09.
  • Rounds candidate buy and sell triggers to the selected multiple, with ₹0.05 as the interface default.
  • Uses Streamlit session state and callbacks to calculate share count from amount, or amount from share count.
  • Rejects non-numeric and non-positive price input with visible feedback.

Engineering decisions

Keep the rule visible

The calculation is a short, inspectable Python function rather than an opaque service. A reviewer can see the thresholds, rounding operation, and fallback behavior directly in `app.py`.

Separate calculation from order execution

The application does not connect to a brokerage account, place a trade, retrieve a live market price, or collect a credential. The visitor supplies the input and remains responsible for verifying the result.

Use reactive state for paired inputs

Streamlit callbacks track whether shares or amount was edited last, clear the paired field, and calculate the corresponding value with conservative floor operations.

Trade-offs and limits

  • Broker rules and allowed tick sizes can change; the repository contains hard-coded assumptions rather than a versioned authoritative rules source.
  • The public repository contains no automated tests, so boundary values and rounding behavior are not protected against regression.
  • `requirements.txt` names Streamlit without pinning a version, which weakens reproducibility.
  • The configured Streamlit URL redirected to an authentication flow during this review, so this page links to the inspectable source rather than promising an accessible live demo.
  • This utility is not financial advice and does not guarantee that a broker will accept a calculated value.

Evidence

This case study is limited to the public repository, its Python implementation, README, MIT license, and repository metadata reviewed on 2026-07-12. It makes no claim about trading outcomes, usage, accuracy against current broker policy, or production scale.

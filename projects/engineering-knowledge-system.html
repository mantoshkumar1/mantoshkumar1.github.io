Evidence-First Engineering Knowledge System

A static engineering publication paired with a grounded assistant that answers only from reviewed public evidence.

Problem

A portfolio can make claims without making them easy to verify. A generic chatbot creates the opposite risk: confident answers that outrun the evidence. This system was designed to make engineering work fast to inspect while keeping every AI answer inside a public, reviewable knowledge boundary.

Architecture

Components and controls

  • GitHub Pages serves the static portfolio, project case studies, Insights, résumé, newsletter, and accessible interaction layer.
  • Markdown in Git is the source of truth for Ask Mantosh. GitHub Actions authenticates indexing with short-lived OIDC credentials.
  • Cloudflare Worker validates requests, enforces quotas, handles deterministic greetings and navigation, and coordinates retrieval.
  • D1 FTS5 and Vectorize provide lexical and semantic retrieval; reciprocal-rank fusion combines the results.
  • Workers AI receives only selected public evidence and returns structured answers with approved citations.

Engineering decisions

  • Evidence before generation. If retrieval finds no sufficiently grounded source, the system declines instead of answering from model memory.
  • Whole-site coverage without false evidence. Projects, Insights, experience, capabilities, skills, achievements, and public profile facts map to reviewed knowledge. Home, indexes, résumé, contact, newsletter, and accessibility requests navigate directly.
  • Private by default. The public client holds no provider credential, and a visitor can export the visible conversation as a local text file without internal identifiers or hidden response metadata.
  • Fail closed. The mandatory rate limiter, bounded input, exact-origin CORS, public-visibility filtering, and citation allowlist protect the public boundary.
  • Automate release confidence. Static audits, Worker evaluation, desktop and mobile browser journeys, WCAG scans, asset budgets, and revision-aware production smoke checks guard publication.

Evaluation evidence

A versioned offline suite exercises the production Worker entry point with controlled D1, Vectorize, and Workers AI bindings—without calling the live service or consuming production quota.

  • 528 labelled cases include 68 focused regressions plus 460 recruiter, student, curious-visitor, colleague, and founder questions.
  • 9,100 objective assertions check response contracts, evidence selection, citation safety, navigation, formatting, output bounds, and appropriate retrieval or generation.
  • Weaknesses became tests. The first persona run passed 434 of 519 cases; routing and evidence boundaries were strengthened, then knowledge-backed profile and whole-site navigation regressions were added.
  • Required release gate. Pull requests fail when a case breaks, committed evidence drifts, or a public page has no Ask Mantosh role.

Boundary: controlled fixtures do not measure live Vectorize recall, human preference, user satisfaction, or organizational impact.

Evidence and limits

The implementation, public knowledge corpus, automated tests, release gates, and production-state documentation are available in the repository for inspection. Fourteen browser checks cover seven visitor journeys in desktop and mobile Chromium; failure artifacts support review without pretending screenshots are pixel-perfect approvals.

Current limits: GitHub Pages rather than a custom domain, Cloudflare free-tier quotas, no declared staging environment, formal accessibility conformance audit, human-rated or live-production retrieval benchmark, cross-browser automation, or production Core Web Vitals trace.
